## 變更摘要
將整個 app 的預設語言改為英文：前端 SPA 新增完整 i18n 系統與即時語言 toggle（英文預設、可切換繁體中文，狀態持久化），公開永久連結頁 chrome 改為英文，使用者面向文件（README / demo README / CHANGELOG）以英文為主並各補一份 zh-TW 譯本、雙向語言連結。內部開發文件（specs/、docs/plans/、.superpowers/、temp/）與 `CLAUDE.md`（給 Claude 的指示檔）刻意維持原樣，不在本次範圍。

## 修改的檔案

### 前端 i18n（核心）
- `public/index.html` — 新增 `I18N` 字典（`en` / `zh-TW` 各 82 鍵，含 `agentExamples` 陣列）、`t()` 查表、`tf()` `{token}` 插值、`applyLang()`；語言 toggle 鈕 `#btn-lang`（緊接 theme 鈕，EN 模式顯示「中文」、zh 模式顯示「EN」）；所有靜態 UI 加 `data-i18n` / `data-i18n-ph` / `data-i18n-title` / `data-i18n-html` 屬性；所有動態字串（toast / confirm / el() 產生的按鈕 / 空狀態 / 提案與完成訊息 / 字元數）改走 `t()` / `tf()`；預設語言 `en`，狀態存 `localStorage['md-memo-lang']`，並透過 `window.__i18nRerender` 讓即時切換重渲染 agent 面板動態內容。

## 技術細節

- **執行方式**：以多 agent workflow 並行完成（i18n 字典抽取 → 前端實作 + permalink + 3 份文件翻譯並行 → contract-reviewer + i18n reviewer 驗證），再由主流程修正與瀏覽器實測。
- **修掉一個 runtime 致命 bug**（i18n reviewer 抓到、`node --check` 無法偵測）：`renderSessionList` 的 `for...of` 迴圈內以 `const t` 命名時間字串，遮蔽了全域翻譯函式 `t()`，導致 `t('agentSessionToMemo')` 拋 `TypeError`，使已存 session 清單載入即靜默變空白（錯誤被 `loadSessions` 的 `catch {}` 吞掉）。修法：迴圈內時間變數改名 `timeStr`。
- **i18n 機制選擇**：靜態文字用 `data-i18n*` 屬性 + `applyLang()` 統一套用；動態文字用 `t()`/`tf()`；即時切換時 `applyLang()` 重跑靜態套用並呼叫 `__i18nRerender` 註冊的重渲染函式（範例 prompt、送出紀錄、已存 session），避免只更新到靜態文字。
- **契約遵守**：未動 `__BASE_PATH__` placeholder、tags 末行慣例與 `parseTags()`；SPA 與 permalink 兩套渲染環境分別處理，互不影響（contract-reviewer 驗證無違反）。
